# md-review

変更された Markdown ファイルを `mo`（ブラウザプレビュー）で可視化しながら、
`difit` スキルでユーザーに差分のレビューを求める。

## Step 1. 対象の特定

`$ARGUMENTS` を解析する:

**`.md` ファイルパスが含まれる場合**
指定されたファイルを mo で開く対象とする。

**コミット・ブランチが含まれる場合**
mo 用の .md ファイルは `git diff --name-only <ref> -- '*.md'` で取得し、
difit にはその引数をそのまま渡す。

**引数なしの場合**
変更された .md ファイルを取得する:

```bash
git status --short -- '*.md' 2>/dev/null | awk '{print $NF}'
```

対象の .md ファイルが 1 件もない場合はユーザーに報告して停止する。
Git 管理外ディレクトリの場合は直接ファイルパスをユーザーに確認する。

## Step 2. mo でブラウザプレビューを起動

対象ファイルをまとめて渡す:

```bash
mo <file1.md> <file2.md> ...
```

複数ファイルは一コマンドで渡す（mo は同一サーバーにまとめて追加する）。
起動後は `http://localhost:6275` をユーザーに伝える。

## Step 3. difit スキルでレビューを依頼する

`difit` スキルを呼び出す。渡す引数は次の通り:

| `$ARGUMENTS` の内容 | difit に渡す引数 |
|---|---|
| コミット・ブランチを含む | `$ARGUMENTS` の値 |
| `.md` ファイルパスのみ / 引数なし | `.`（未コミット変更全体）|

difit スキルはユーザーのレビューコメントの受け取りと対応まで担う。
